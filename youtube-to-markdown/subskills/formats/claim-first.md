# Claim-First Bullets

Status: archive
Content type: INTERVIEW (alternative)

Long informative claim heading + evidence bullets. Lighter than Concept Card — no core idea or implication sentences. Scans fast but gives no reading depth choices between heading and bullets.

## Template

```
### [Claim as long informative heading]
- [evidence bullet, front-loaded]
- [evidence bullet]
- [evidence bullet]
```

## Example

```
### Running 5-10 parallel sessions beats going deep on one task
- Boris kicks off tasks from terminal, phone, and web simultaneously
- "Tending to your Claudes" — bounce between sessions, unblock each
- Parallel execution offsets slower per-task speed
- Real example: Slack-message engineers whose status columns are empty, go get coffee
```

## Rules

- Heading: one complete claim sentence, informative enough to decide whether to read bullets
- Bullets: 3-5 per section, front-loaded
- No separator between sections (unlike concept-card)
- No core idea or implication sentences
