# DONE: IMAP Session Caching

## Intent

Reduce latency, IMAP server load, and token waste by caching folder listings and message lists with persistent connections.

## Goals

1. Connection keepalive (5 min idle timeout) instead of connect/disconnect per operation
2. Cache folder listings (until MCP shutdown)
3. Cache message lists with IMAP metadata validation (UIDVALIDITY, UIDNEXT, EXISTS)
4. Single entry point for all IMAP operations via `AccountSession`

## Constraints

- Depends on flag management being implemented first
- In-memory cache only, no file persistence
- Cache per account
- Cache lives until MCP shutdown (small footprint, validated on use)
- Connection closes after 5 min idle, reopens on next request
- Test against Drafts folder, not INBOX

## Architecture

### Data Structures

```python
@dataclass
class FolderCache:
    folders: list[dict]          # [{name, flags}, ...]
    fetched_at: float            # for logging/debugging

@dataclass
class MessageListCache:
    messages: list[dict]         # [{uid, subject, from, date, flags}, ...]
    uidvalidity: int             # invalidation check
    uidnext: int                 # new message detection
    exists: int                  # message count

class AccountSession:
    account: str
    connection: IMAPClient | None
    last_activity: float         # for connection idle timeout
    folder_cache: FolderCache | None
    message_cache: dict[str, MessageListCache]  # folder -> cache
```

### Session Management

```python
CONNECTION_IDLE_TIMEOUT = 300  # 5 minutes
_sessions: dict[str, AccountSession] = {}

def get_session(account: str | None = None) -> AccountSession:
    """Get or create session for account."""
    account = account or get_default_account()
    if account not in _sessions:
        _sessions[account] = AccountSession(account)
    return _sessions[account]
```

### Connection Management

```python
class AccountSession:
    def get_connection(self) -> IMAPClient:
        now = time.time()

        if self.connection:
            if now - self.last_activity > CONNECTION_IDLE_TIMEOUT:
                self._close_connection()
            else:
                try:
                    self.connection.noop()
                except:
                    self._close_connection()

        if not self.connection:
            self.connection = _create_imap_connection(self.account)

        self.last_activity = now
        return self.connection

    @contextmanager
    def connection(self) -> IMAPClient:
        try:
            conn = self.get_connection()
            yield conn
            self.last_activity = time.time()
        except (IMAPClientError, ConnectionError, socket.error):
            self._close_connection()  # Keep caches, only reset connection
            raise
```

### Cache Validation

```python
class AccountSession:
    def get_folders(self) -> list[dict]:
        if self.folder_cache:
            return self.folder_cache.folders

        conn = self.get_connection()
        folders = conn.list_folders()
        self.folder_cache = FolderCache(
            folders=[{"name": to_str(name), "flags": [to_str(f) for f in flags]}
                     for flags, _, name in folders],
            fetched_at=time.time()
        )
        return self.folder_cache.folders

    def get_messages(self, folder: str, limit: int) -> list[dict]:
        conn = self.get_connection()
        conn.select_folder(folder)

        status = conn.folder_status(folder, ['UIDVALIDITY', 'UIDNEXT', 'MESSAGES'])
        uidvalidity = status[b'UIDVALIDITY']
        uidnext = status[b'UIDNEXT']
        exists = status[b'MESSAGES']

        cached = self.message_cache.get(folder)
        if cached and cached.uidvalidity == uidvalidity and \
           cached.uidnext == uidnext and cached.exists == exists:
            return cached.messages[:limit]

        messages = self._fetch_message_list(conn, folder, limit)
        self.message_cache[folder] = MessageListCache(
            messages=messages,
            uidvalidity=uidvalidity,
            uidnext=uidnext,
            exists=exists
        )
        return messages
```

### Cache Invalidation on Write Operations

| Operation | Cache action |
|-----------|--------------|
| Flag change | Update message flags in-place |
| Move message | Invalidate source + destination folder caches |
| Delete message | Invalidate folder cache |
| Create/modify draft | Invalidate Drafts folder cache |

```python
def invalidate_folder_cache(account: str, folder: str):
    session = _sessions.get(account)
    if session:
        session.message_cache.pop(folder, None)
```

### Integration

Remove `imap_connection()` context manager. All operations via `get_session()`:

```python
# Cached operations
def list_folders(account: str = None) -> list[dict]:
    session = get_session(account)
    return session.get_folders()

# Non-cached operations
def create_draft(..., account: str = None):
    session = get_session(account)
    with session.connection() as conn:
        # Draft operations...
```

## Acceptance Criteria

1. Second `list_folders` call returns instantly (cache hit)
2. Second `list_messages` call with no changes returns instantly (cache hit)
3. New message in folder detected via UIDNEXT, triggers refetch
4. Flag operation updates cache in-place
5. Connection reused within 5 min window
6. Connection recreated after 5 min idle
7. Connection error does not clear cache
8. All tests pass

## Lessons Learned

### Implementation Insights

1. **Context manager naming**: `connection()` method name conflicted with `connection` attribute. Renamed to `connection_ctx()` for clarity.

2. **Circular import avoidance**: `session.py` needs `get_credentials()` from `imap_client.py`. Solved with lazy import inside function (`from imap_client import get_credentials`).

3. **Migration requires consistency**: Code review caught that `create_draft`, `modify_draft` and `modify_flags` still used old `imap_connection()` context. All operations must migrate together, otherwise connection keepalive benefit is partially lost.

4. **Keeping `imap_connection()`**: Plan was to remove entirely, but it makes sense for `test_connection()` standalone utility. Documented the intended use.

5. **Message cache validation is not free**: Second `get_messages()` call takes ~37ms (vs. 330ms first), because `folder_status()` call is needed for UIDVALIDITY/UIDNEXT/EXISTS check. Acceptable trade-off vs. stale data.

6. **Test-implementation consistency**: Tests used `uid` key, implementation used `id`. Small but confusing inconsistency - worth checking in code review.
